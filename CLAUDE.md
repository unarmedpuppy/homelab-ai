## Design Context

### Users
Solo operator (Josh). Primary context: checking on the AI fleet while doing other things — not sitting down to "use" the dashboard. Glances to verify status, dispatches jobs, reviews sessions and stats. The interface should surface what matters immediately without demanding attention.

### Brand Personality
**Technical. Intentional. Controlled.**

This is infrastructure tooling, not a product. It should feel like something a skilled engineer built for themselves: opinionated, efficient, and uncompromising about clarity. The AoE2 Command page sets the tone — playful in concept, precise in execution.

### Aesthetic Direction
**Linear/Vercel dark** — minimal dark UI, tight spacing, clean sans-serif where appropriate (Berkeley Mono for data and code), precise 1px borders, no decorative noise.

**Reference feel**: Linear, Vercel dashboard, GitHub dark mode
**Anti-references**: Generic Tailwind UI SaaS, enterprise data tables, emoji-laden UIs, glow-heavy "cyberpunk" styling, anything that looks AI-generated by default

Dark mode only. No light variant needed.

### Design Principles

1. **Data over decoration** — Every visual element must earn its place. If it doesn't communicate something, remove it. No gradients, glows, or motion for its own sake.

2. **Mobile-first, always** — The existing system has mobile support; all new work starts there and expands up. Touch targets ≥44px. No desktop-only assumptions.

3. **Density with breathing room** — Command centers are dense, but not claustrophobic. Tight spacing within components, deliberate whitespace between them. Hierarchy through weight and color, not size.

4. **Status is the primary language** — Green/yellow/red/blue carry meaning throughout. Never use semantic colors decoratively. A green badge means online; it should never mean "we like this feature."

5. **No emojis. No vibecoded UI.** — Labels are words. Icons are SVG. The interface looks like it was designed, not generated. Typography does the work.

### Technical Constraints
- React 19 + TypeScript + Vite
- Tailwind CSS 4 (utility-first, PostCSS)
- Berkeley Mono as the monospace font — use it for all data, values, IDs, code, and terminal-adjacent content
- Custom CSS variables in `retro-theme.css` (dark slate palette: `#0f172a` bg, `#1e293b` surface)
- Component library: `src/components/ui/` — extend, don't replace
- Recharts for data viz, Phaser 3 for Command page (isolated, don't style-match)
